# Parcel

- Dev Build – Creates a fast development build for quick testing and debugging.
- Local Server – Provides a built-in server to run the app locally without extra setup.
- HMR (Hot Module Replacement) – Updates only the changed module instantly without
  refreshing the page.
- File Watching Algorithm (C++) – Uses an efficient C++-based watcher to detect file changes
  quickly.
- Caching – Stores build results to speed up subsequent builds.
- Image Optimization – Automatically optimizes images to improve performance.
- Minification – Removes unnecessary code to reduce file size.
- Bundling – Combines multiple files into optimized bundles.
- Compression – Compresses files for faster network transfer.
- Code Splitting – Loads only required code to reduce initial load time.
- Differential Bundling – Creates separate bundles for modern and older browsers.
- Diagnostics – Shows clear and helpful error messages.
- Error Handling – Detects and reports errors early for easier debugging.
- Consistent Hashing – Generates stable file names to improve browser caching.
- HTTPs - Enables secure, encrypted communication between the browser and server during
  development.
- Tree Shaking – Removes unused code to reduce bundle size and improve performance.
- Different dev and prod bundles
